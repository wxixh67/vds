    容器服务 ¶

    • 盘点那些免费的 Docker 容器（1）
    • 盘点那些免费的 Docker 容器（2）

    基本来自上面的帖子，下面是我自己体验过的评价：

    • Claw cloud
      • 注册需要 180 天以上的 github 账号最佳，其余情况似乎有其他限制？无需绑卡
      • 每个月送 5 $ 额度，部署一两个小服务相当于免费了；存储一体化
      • 相对不是那么稳定，使用 cf 绑定自定义域名非常非常非常慢……
    • Koyeb
      • 注册较随意，不用绑卡
      • 免费部署一个低配服务；部署较快，界面清爽
      • 会重启容器，如果有存储需求需要额外的数据库服务（见下）；免费版不能自定义域名
    • Render
      • 注册较随意，（疑似）需要绑卡
      • 按照免费配置可部署任意多容器，所有容器共享 750 h；部署较快
      • 会重启容器，如果有存储需求需要额外的数据库服务（见下）

    文档（PDF 为主）¶

    • 免费在线文档转换，pdf 操作
      • ILovePdf
      • XPdf
      • pdnob
      • smallpdf
    • 离线转换器
      • “离线转换器与在线转换器相反，它是为了最大限度地保护您的隐私，因为所有文件都是在本地转换而不是云服务器。离线转换器的另一个优点是每个文件大小可以达到 2GB，而很少有在线转换器支持大于 100MB 的文件。”
    • pandoc
      • a universal document converter
    • 英文论文翻译
      • 幻觉翻译
        • 来自消耗 3 亿 token，我用大模型翻译了 1 万篇 arXiv 论文
      • ChatPaper
        • 可以免费使用学术版 GPT 网页非盈利版
          • 针对 arXiv 比较厉害的一个功能是，根据论文摘要链接，获取 tex 源码，对其翻译后再编译，所以翻译后看起来比直接翻译 pdf 好很多
        • 示例教程
        • zh_Traced ICSE 24 是一个成功例子
      • 有道在线翻译
        • 有道的翻译也不错，其 arXiv 论文翻译排版效果也挺好
      • 沉浸式翻译
        • 在阅读 Html 版本时可以搭配使用
        • Tips:（以 https://arxiv.org/abs/1502.03808 为例）​
          • 替换「x」为「5」跳转到 HTML 版本阅读​，搭配沉浸式翻译使用：https://ar5iv.org/abs/1502.03808
          • 替换「ar」为「alpha」跳转到 alphxiv 处理 https://alphaxiv.org/abs/1502.03808
          • 替换「v」为「w」跳转到 txyz 处理​：https://arxiw.org/abs/1502.03808
      • paf2zh from PDFMathTranslate

    X 床 ¶

    取图“床”之意，表示能够暂时借助以在线存储 / 传播一些东西。

    • imgbb or smms
      • 免费图床，敏感图片就最好自己搭建图床了。
    • godbolt
      • 免费“代码床”，因为右上角 share 可以获得 保留代码 的链接；
      • 在线编辑器，可以同时切换不同版本编译器运行代码；
      • 支持展示编译内容，例如。
      • 其他“知名”代码床：Github gist or LeetCode/playground or One Compiler
